Event and Role Annotation Instructions

Updated instructions as of Jan 28, 2025 (*new annotated examples added).

Overview

In this task, we give you a highlighted word or phrase in a Passage Text. The highlighted phrase refers to some kind of situation -- for example, a dynamic event (like running or kicking) or a state of being (like happiness or redness). We also give you the Source Text from which this Passage was taken. The Passage text is essentially a summary of some situation from the Source Text. We then ask you to do the following:

  1. identify the "event type" of the highlighted phrase in the Passage Text
    Note: This is already pre-highlighted for you in this task based on an earlier task performed by workers. If you still strongly feel that the pre-highlighted event type is not correct, feel free to change it.
  2. identify different roles of event in (i) in the Passage Text
  3. identify the different roles of the same event in Source Text

If you prefer watching a video, you can view it here: Annotation Task Example

1. Identifying the Event Type

Let us look at the task through a running example. We have highlighted one word that refers to a situation (or event) in the Passage Text as shown below:

He was hired as a Research Scientist by Microsoft.

Your first task is to identify what type of event is highlighted in the Passage Text from a list of options. For example, for "hired" you might see choices like:

For our running example, Hiring is the correct event type.

2. Identifying the Roles in Passage Text

Once you finalize the event type, start annotating the roles of that event in the Passage Text. Roles are different participants of the event. For example, the "Hiring" event type has roles like "Employee", "Employer", "Position", etc.

2.1 Role Annotation Examples

2.1.1 Single Word Role: "Employee"

Select by clicking and dragging over the word(s):

He was hired as a Research Scientist by Microsoft.

Here, "He" is the Employee role.

2.1.2 Multiple Word Role: "Position"

Select multiple words for a role when needed:

He was hired as a Research Scientist by Microsoft.

Here, "as a Research Scientist" is the Position role.

2.1.3 Missing Role: "Task"

If no text mentions a role, select "No candidate found for the current role"

Example: In "He was hired as a Research Scientist by Microsoft", there's no mention of what tasks he'll perform.

2.1.4 Valid Candidates

The system shows valid overlapping candidates you can select from. Choose the most appropriate one.

3. Identifying the Roles in Source Text

After identifying roles in the Passage Text, toggle to Source Text and identify roles there. The rules are the same, but the event phrase isn't highlighted in the source text. You'll need to identify roles for the same event that was highlighted in the passage.

The Source Text often contains more detailed information, so you may find roles that weren't present in the Passage Text.

4. Handling Edge Cases in Selecting Spans

If you can't select the whole text span you need:

  1. Keep a partial span selected and click the "plain text" button
  2. Copy the complete desired span from the plain text version
  3. Click back to the original text view
  4. Paste into the type-in bar at the top of candidates list
  5. Click "Add Custom Candidate"
  6. Select your newly added candidate

5. General Rules for Annotating Roles

5.1 Use the most informative phrase

Select in order of preference:

  1. Names (e.g., "John Smith", "United States")
  2. Descriptive mentions (e.g., "the 32-year-old man", "the tallest building")
  3. Pronouns (e.g., "he", "she", "they", "it")
5.2 Add extra roles when needed

If multiple participants fill the same role (e.g., multiple employees being hired), use the "Add Role" dropdown to create additional instances of that role (e.g., Employee__2, Employee__3).

5.3 Making valid candidates

If your selection isn't listed as a valid candidate, click "Add currently selected text as a valid candidate".

5.4 Keyboard shortcuts

Press "?" to view all keyboard shortcuts:

  • 1-9: Select candidate from list
  • n: No candidate found
  • Tab: Next role
  • Shift+Tab: Previous role
  • ?: Show shortcuts
